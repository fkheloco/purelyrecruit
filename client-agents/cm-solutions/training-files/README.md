## TRAINING PROTOCOLS

### Adding New Training Materials

When new training is conducted or materials are created:

1. **Document the Training**
   - What was trained?
   - When?
   - Why?
   - Key learnings

2. **Organize & Archive**
   - Create dated folder: `YYYY-MM-Q#-[topic]-training/`
   - Include training materials (documents, examples, etc.)
   - Include summary document

3. **Update Main Knowledge Base**
   - Integrate new information into main files
   - Update Claude.md if core instructions changed
   - Update relevant knowledge base files
   - Update README with new context

4. **Tag & Reference**
   - Tag materials with topic areas
   - Cross-reference in related documents
   - Create index if large training package

## UPDATING KNOWLEDGE BASE FILES

### When Knowledge Changes

**For Company Information:**
- Update `/knowledge-base/company/company-overview.md`
- Update `/knowledge-base/company/leadership-team.md` if team changes
- Update `/knowledge-base/company/products-services.md` if services change

**For Sales & Positioning:**
- Update `/knowledge-base/sales/value-propositions.md`
- Update `/knowledge-base/clients/ideal-customer-profile.md`
- Update `/knowledge-base/clients/project-types.md`

**For Processes:**
- Update `/knowledge-base/processes/proposal-process.md`
- Update `/knowledge-base/processes/recruitment-process.md`

**For Brand & Communications:**
- Update `/brand/guidelines/voice-and-tone.md`
- Update `/Claude.md` if core agent instructions change
- Update `/README.md` if system-level changes

### Update Process

1. **Identify the Change**
   - What information needs updating?
   - Which file(s) are affected?
   - Is this a major strategic change?

2. **Document the Change**
   - What's changing and why?
   - When does it take effect?
   - Who should know about it?

3. **Update the Files**
   - Make updates to relevant knowledge base files
   - Update cross-references if applicable
   - Maintain consistent tone and formatting

4. **Archive the Old Version** (If Needed)
   - Keep dated backup of major changes
   - Note date of update in file header
   - Document reason for update

5. **Notify Team** (If Major Change)
   - Communicate significant updates
   - Highlight key changes
   - Request feedback if needed
